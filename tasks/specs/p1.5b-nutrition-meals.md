# P1.5-B — Nutrition Meal Tracking

## Overview

Phase A (daily totals) is already live. This phase adds meal-level granularity:
athletes log individual meals (breakfast, lunch, dinner, snack) by picking food items
from a searchable database. Daily totals are auto-computed from meal entries.

Trainers with an accepted coaching relationship can view an athlete's meal history.

## Dependencies

**P1.5-A must be complete** (NutritionGoal + DailyNutritionLog already exist and are live).

When B is complete, `DailyNutritionLog` entries should still be usable as manual overrides —
the auto-compute from meals should be opt-in, not forceful.

### 6. New endpoint file — `src/TrackMe.Api/Endpoints/MealEndpoints.cs`

```csharp
public static IEndpointRouteBuilder MapMealEndpoints(this IEndpointRouteBuilder app)
{
    var group = app.MapGroup("/api/nutrition").RequireAuthorization();

    // Food items
    group.MapGet("/foods", SearchFoods);
    group.MapPost("/foods", CreateFood);
    group.MapGet("/foods/{id:guid}", GetFoodById);

    // Meals
    group.MapPost("/meals", CreateMeal);
    group.MapGet("/meals/me", GetMyMeals);           // ?date=YYYY-MM-DD
    group.MapGet("/meals/{athleteId:guid}", GetAthleteMeals); // trainer view, ?date=
    group.MapDelete("/meals/{id:guid}", DeleteMeal);

    // Meal entries
    group.MapPost("/meals/{id:guid}/entries", AddEntry);
    group.MapPatch("/meals/{id:guid}/entries/{entryId:guid}", UpdateEntry);
    group.MapDelete("/meals/{id:guid}/entries/{entryId:guid}", DeleteEntry);

    return app;
}
```

**Handler logic:**

`SearchFoods` — `GET /api/nutrition/foods?q=&page=&pageSize=`
- Search `food_items` by name (ILIKE `%q%`)
- Returns global items + items created by the calling user
- Returns paginated `FoodItemDto[]`

`CreateFood` — Trainer or athlete can create custom food items (`IsGlobal = false`, `CreatedByUserId = caller`)
- Admin can set `IsGlobal = true`
- Body: `CreateFoodItemRequest`
- Returns 201 `FoodItemDto`

`CreateMeal` — Athlete (or trainer on behalf of their athlete) creates a meal container
- For athlete: `AthleteId` in request must match own profile
- For trainer: must have accepted coaching relationship with the athlete
- Returns 201 `MealDto` (with empty `Entries`)

`GetMyMeals` — Returns `DayMealsDto` for the given date (query param `?date=YYYY-MM-DD`, defaults to today)
- Loads all meals + entries + food items for that athlete + date
- Computes daily totals from entries

`GetAthleteMeals` — Trainer view. Same as above but for another athlete. Requires accepted coaching relationship.

`DeleteMeal` — Owner (athlete) or admin only. Cascades to entries.

`AddEntry` — Adds a food item to a meal.
- Compute macros at save time: `caloriesKcal = foodItem.CaloriesPer100g * amountGrams / 100` (same for others)
- Returns updated `MealDto`

`UpdateEntry` — Change `AmountGrams`. Recompute macros in place. Returns updated entry.

`DeleteEntry` — Owner or admin. Returns 204.

**Macro computation helper** (private static):
```csharp
private static (decimal cal, decimal prot, decimal carbs, decimal fat) ComputeMacros(
    FoodItem food, decimal amountGrams)
{
    var ratio = amountGrams / 100m;
    return (
        Math.Round(food.CaloriesPer100g * ratio, 2),
        Math.Round(food.ProteinPer100g * ratio, 2),
        Math.Round(food.CarbsPer100g * ratio, 2),
        Math.Round(food.FatPer100g * ratio, 2)
    );
}
```

**DayMealsDto builder** (private static):
```csharp
private static DayMealsDto BuildDayDto(DateOnly date, List<Meal> meals) => new(
    date,
    meals.OrderBy(m => m.MealType).Select(ToMealDto).ToList(),
    meals.SelectMany(m => m.Entries).Sum(e => e.CaloriesKcal),
    meals.SelectMany(m => m.Entries).Sum(e => e.ProteinG),
    meals.SelectMany(m => m.Entries).Sum(e => e.CarbsG),
    meals.SelectMany(m => m.Entries).Sum(e => e.FatG));
```

**Access pattern** — same as NutritionEndpoints: use `ResolveAthleteAsync` for self, `HasAcceptedRelationshipAsync` for trainer access.
