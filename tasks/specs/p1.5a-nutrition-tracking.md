# P1.5-A — Nutrition Tracking (MVP)

## Overview

Trainer sets a daily macro goal for an athlete (calories, protein, carbs, fat).
Athlete logs their daily totals each day.
Both sides can see adherence — athlete sees a daily card on dashboard, trainer sees
a 30-day compliance graph in the athlete detail view.

This is Phase A (daily totals only). Phase B (meal + food database) is a separate spec.

## Tech stack context

- **API**: ASP.NET Core 10 Minimal API, EF Core 10, PostgreSQL 16
- **Frontend**: React 18 + Vite SPA
- **Repos**: `TrackMe-Api` · `TrackMe-Web`

## Dependencies

No external dependencies. Can be built standalone.

### 5. New endpoint file — `src/TrackMe.Api/Endpoints/NutritionEndpoints.cs`

```csharp
using System.Security.Claims;
using Microsoft.AspNetCore.Builder;
using Microsoft.AspNetCore.Http;
using Microsoft.AspNetCore.Routing;
using Microsoft.EntityFrameworkCore;
using TrackMe.Api.Data;
using TrackMe.Api.Models;
using TrackMe.Api.Services;

namespace TrackMe.Api.Endpoints;

public static class NutritionEndpoints
{
    public static IEndpointRouteBuilder MapNutritionEndpoints(this IEndpointRouteBuilder app)
    {
        // Goals
        app.MapPost("/api/nutrition/goals", SetGoal).RequireAuthorization();
        app.MapGet("/api/nutrition/goals/{athleteId:guid}", GetGoal).RequireAuthorization();
        app.MapPatch("/api/nutrition/goals/{id:guid}", UpdateGoal).RequireAuthorization();

        // Logs
        app.MapPost("/api/nutrition/logs", LogDay).RequireAuthorization();
        app.MapGet("/api/nutrition/logs/me", GetMyLogs).RequireAuthorization();
        app.MapGet("/api/nutrition/logs/{athleteId:guid}", GetAthleteLogs).RequireAuthorization();
        app.MapPatch("/api/nutrition/logs/{id:guid}", UpdateLog).RequireAuthorization();

        // Summary (dashboard card)
        app.MapGet("/api/nutrition/summary", GetSummary).RequireAuthorization();

        return app;
    }
    // ... handlers below
}
```

**Handler logic:**

`SetGoal` — Trainer (or dual-role) only. Deactivates previous active goal for the athlete, creates new one.
```
POST body: SetNutritionGoalRequest
Returns: 201 NutritionGoalDto
Access: Trainer with accepted coaching relationship OR athlete setting own goal
```

`GetGoal` — Returns active goal for athlete. Trainer or athlete (coaching relationship required for trainer).
```
GET /api/nutrition/goals/{athleteId}
Returns: NutritionGoalDto or 404
```

`UpdateGoal` — Trainer or goal owner (athlete). Updates calories/macros.
```
PATCH body: UpdateNutritionGoalRequest
Returns: 200 NutritionGoalDto
```

`LogDay` — Athlete only. Upsert logic: if a log for `Date` already exists for this athlete, update it instead of inserting.
```
POST body: LogNutritionRequest
Returns: 200 DailyNutritionLogDto (upsert)
Access: athlete or dual-role with athlete profile
```

`GetMyLogs` — Athlete's own logs. Optional query params: `from` (DateOnly), `to` (DateOnly). Default: last 30 days.
```
GET /api/nutrition/logs/me?from=2024-01-01&to=2024-01-31
Returns: DailyNutritionLogDto[]
```

`GetAthleteLogs` — Trainer view. Same date-range params. Requires accepted coaching relationship.
```
GET /api/nutrition/logs/{athleteId}?from=&to=
Returns: DailyNutritionLogDto[]
```

`UpdateLog` — Athlete updates own log. Updates all macro fields + notes.
```
PATCH body: UpdateNutritionLogRequest
Returns: 200 DailyNutritionLogDto
```

`GetSummary` — Returns active goal + today's log + last 7 days logs. Used for the dashboard card.
```
GET /api/nutrition/summary
Returns: NutritionSummaryDto
```

**Access helper** (copy/adapt from ProgressPhotoEndpoints.cs):
- Athlete resolve: by `profileId` from JWT claims (if Athlete role) or by email lookup (if Trainer role with dual-role profile)
- Trainer access gate: `EndpointHelpers.HasAcceptedRelationshipAsync(db, trainerId, athleteId)`
- Forbidden: `EndpointHelpers.Forbidden("message")`

### 3. `NutritionView.jsx` — new file at `src/views/NutritionView.jsx`

Athlete perspective:

**Sections:**
1. **Today card** — 4 progress bars (Calories, Protein, Carbs, Fat) showing logged vs goal. "Log Today" button opens modal if not yet logged today; "Update" if already logged.
2. **Log modal** — 4 number inputs (kcal, g, g, g) + notes textarea. Pre-fills today's log if it exists.
3. **30-day adherence grid** — one square per day for last 30 days; green if calories >= 80% of goal, yellow 60-79%, red < 60%, grey = no log. Hover shows date + values.
4. **No goal state** — show message "Trainer has not set a nutrition goal yet."

**Props:** `currentUser`, `t`
